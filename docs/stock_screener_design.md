## Problem Statement

Rainier currently scrapes QU100 money flow data (top/bottom 100 stocks, capital flow direction, long/short dominance) and stores it in TimescaleDB. But this data sits unused beyond raw storage. The missing piece: **automatically screening QU100 stocks to find those about to start or just starting an uptrend**, combining money flow signals with chart pattern detection from 蔡森's "多空转折一手抓" methodology.

The goal is a 3-layer pipeline:
1. **Money Flow Filter** — QU100 data identifies stocks where institutional money is flowing in
2. **Sector Trend Filter** — Aggregate sector-level momentum to ride sector rotations
3. **Technical Pattern Detection** — 蔡森's 12 chart patterns detect reversal/continuation setups

Output: A ranked list of stocks with buy/sell signals, entry prices, stop losses, and price targets.

## Demand Evidence

Rainier is already generating value for futures trading. The QU100 scraper runs 4x daily collecting money flow data on 200 stocks (top 100 + bottom 100). This data is persisted but not analyzed. The book provides a complete, codifiable methodology for detecting trend reversals that maps directly to the QU100 stock universe.

## Target User & Narrowest Wedge

Edison (solo trader) — the narrowest wedge is: each morning, get a list of 3-5 stocks from QU100 that have money flowing in AND show a bullish chart pattern forming. Replace manual chart scanning of 100 stocks with automated screening.

## Constraints

- Must work with existing QU100 data in TimescaleDB (MoneyFlowSnapshot, StockCapitalFlow tables)
- Stock price data via yfinance (already integrated as data provider)
- Daily chart analysis (weekly for trend confirmation)
- US stock market — no gaps concern within session, but overnight gaps exist
- Volume data available from yfinance (critical for 蔡森's 量价 methodology)
- Must produce actionable output: stock, pattern, entry, SL, target, confidence

## Premises

1. QU100 "Long in" stocks with positive capital flow are the primary candidate pool
2. 蔡森's 12 patterns are codifiable from OHLCV data + volume
3. Daily timeframe is primary; weekly for trend confirmation
4. The 3-layer funnel (money flow → sector → pattern) reduces 100 stocks to 3-5 actionable ideas
5. Volume-price structure (量价结构) is the leading indicator — patterns without volume confirmation are lower confidence

## Detailed Design

### Layer 1: Money Flow Screener (`analysis/stock_screener.py` — NEW)

Query QU100 data to find stocks with bullish money flow signals.

**Screening criteria (all must be true):**

```python
@dataclass
class MoneyFlowSignal:
    symbol: str
    rank: int                    # QU100 ranking (1-100, lower is stronger)
    rank_change: int             # Daily rank change (positive = improving)
    long_short: str              # "Long in" required for bullish
    capital_flow_direction: str  # "+" required for bullish
    days_in_top100: int          # Consecutive days in top 100
    sector: str
    industry: str
    signal_strength: float       # 0-1 composite score
```

**Scoring formula:**
- `long_short == "Long in"` → base score 0.5
- `capital_flow_direction == "+"` → +0.2
- `rank <= 30` → +0.15 (top tier)
- `rank_change > 0` (improving) → +0.1
- `days_in_top100 >= 3` (sustained) → +0.05

**Query logic (two-step — both are REQUIRED):**

Step 1: Get latest QU100 ranking + long/short status from MoneyFlowSnapshot:
```sql
SELECT s.symbol, s.id as stock_id, m.rank, m.daily_change, m.long_short, m.sector, m.industry
FROM money_flow_snapshots m
JOIN stocks s ON s.id = m.stock_id
WHERE m.ranking_type = 'top100'
  AND m.long_short = 'Long in'
  AND m.captured_at = (SELECT MAX(captured_at) FROM money_flow_snapshots)
ORDER BY m.rank ASC;
```

Step 2: For each stock from Step 1, get capital_flow_direction from StockCapitalFlow (this is where "+" inflow lives — NOT on MoneyFlowSnapshot):
```sql
SELECT scf.flow_date, scf.capital_flow_direction, scf.long_short, scf.rank
FROM stock_capital_flow scf
WHERE scf.stock_id = :stock_id
  AND scf.period_type = 'daily'
ORDER BY scf.flow_date DESC
LIMIT 5;
```

Both queries are mandatory. `long_short` comes from Step 1, `capital_flow_direction` comes from Step 2.

A stock scores higher if it has 3+ consecutive days of "+" capital flow AND "Long in" status.

**Output:** List of `MoneyFlowSignal` objects, sorted by `signal_strength` descending. Typically 20-40 stocks pass this filter.

### Layer 2: Sector Trend Analyzer (`analysis/sector_analyzer.py` — NEW)

Aggregate money flow signals by sector to identify sector-level trends.

```python
@dataclass
class SectorTrend:
    sector: str
    long_in_count: int           # Number of "Long in" stocks in this sector
    short_in_count: int          # Number of "Short in" stocks
    net_sentiment: float         # (long_in - short_in) / total
    top_stocks: list[str]        # Best ranked stocks in sector
    trend_direction: str         # "bullish", "bearish", "neutral"
    sector_rank: int             # Rank among all sectors
```

**Logic:**
1. Group all QU100 stocks (top100 + bottom100) by `sector`
2. For each sector, count "Long in" vs "Short in" vs "No dominance"
3. `net_sentiment = (long_count - short_count) / total_count`
4. `trend_direction`:
   - `net_sentiment > 0.3` → "bullish"
   - `net_sentiment < -0.3` → "bearish"
   - else → "neutral"
5. Rank sectors by `net_sentiment` descending

**Sector boost:** Stocks in a bullish sector get +0.1 added to their Layer 1 `signal_strength`. This means a rank-50 stock in a hot sector may outrank a rank-30 stock in a neutral sector.

**Output:** List of `SectorTrend` objects + boosted `MoneyFlowSignal` scores.

```python
    weekly = yf.download(symbols, period="2y", interval="1wk", group_by="ticker")
```

```python
@dataclass
class Neckline:
    price: float        # Neckline price level
    slope: float        # 0 for horizontal, positive for ascending
    touch_points: list[int]  # Bar indices where price touched neckline
    type: str           # "support" or "resistance"

def find_neckline(swing_points: list[SwingPoint], type: str) -> Neckline | None:
    """Connect swing lows (support neckline) or swing highs (resistance neckline).
    Use the principle: more points on a line = more accurate (越多点连线越准确).

    Algorithm: Iterative linear regression with outlier rejection.
    1. Fit a line through all candidate swing points (least squares)
    2. Remove points > neckline_tolerance_pct from the fitted line
    3. Refit with remaining points
    4. A "touch" = price within neckline_tolerance_pct of the fitted line
    5. Require >= 2 touch points for a valid neckline
    For horizontal necklines (W bottom, M top): slope near zero → use mean price.
    For sloped necklines (H&S): slope allowed, but > 30 degree slope is rejected."""
```

#### 3.3 The 12 Pattern Detectors

Each pattern detector returns:

```python
@dataclass
class PatternSignal:
    symbol: str
    pattern_type: str           # "w_bottom", "false_breakdown", "bull_flag", etc.
    direction: str              # "bullish" or "bearish"
    status: str                 # "forming", "confirmed", "target_reached"
    confidence: float           # 0-1, based on volume confirmation + pattern clarity

    # Key price levels
    entry_price: float          # Where to enter
    stop_loss: float            # Where to exit if wrong
    target_wave1: float         # 第一波涨/跌幅满足
    target_wave2: float | None  # 第二波涨/跌幅满足 (if applicable)
    risk_pct: float             # (entry - SL) / entry as percentage
    reward_pct: float           # (target - entry) / entry as percentage
    rr_ratio: float             # reward / risk

    # Pattern components
    neckline: float
    key_points: dict            # Pattern-specific points (e.g., {"left_bottom": 14, "right_bottom": 13})
    volume_confirmed: bool      # 带量突破

    # Timestamps
    pattern_start: date
    pattern_end: date | None    # None if still forming
    breakout_date: date | None
```

**Stop loss convention (all patterns):** Stop loss is always calculated as a percentage below (bullish) or above (bearish) a specific reference level, NOT from the entry price. The reference level is pattern-specific (e.g., neckline, false low). Default `stop_buffer_pct = 0.02` (2% below the reference level) is configurable in settings.yaml.

**Pattern dedup policy:** A pattern is identified by the key `(stock_id, pattern_type, round(neckline, 2))`. If the same key exists in `stock_pattern_signals` with `detected_at` within the last 30 days, UPDATE the existing row (status, confidence may change). Otherwise INSERT a new row. The `new_signals` alert list only fires on INSERT or on status change from "forming" → "confirmed".

**Pattern 1: W Bottom (W底)**
```
Detection:
  1. Find two swing lows at approximately same level (within 3% tolerance)
  2. Find the swing high between them = neckline
  3. If current price breaks above neckline → CONFIRMED
  4. If price is approaching neckline from below → FORMING

Target calculation:
  distance = neckline - avg(bottom1, bottom2)
  target_wave1 = neckline + distance
  target_wave2 = target_wave1 + distance

Entry: At neckline breakout (or at false breakdown recovery for early entry)
Stop loss: Below neckline (5-7%), or below the lower of the two bottoms
Volume: Breakout bar volume > 1.5x average = confirmed
```

**Pattern 2: False Breakdown (破底翻)**
```
Detection:
  1. Price in consolidation zone (identify support level)
  2. Price breaks below support (new low)
  3. Price quickly reverses back above support within 1-5 bars
  4. The false breakdown low becomes the true bottom

Entry: When price recovers above the broken support level
Stop loss: Below the false breakdown low (5-7%)
Target: Neckline breakout of the consolidation range, then W bottom target
Volume: Recovery bar with increasing volume = strong signal
Confidence boost: +0.2 if combined with W bottom structure
```

**Pattern 3: False Breakdown W Bottom (破底翻 W底)**
```
Detection: Combination of Pattern 1 + Pattern 2
  1. W bottom structure forming
  2. Second bottom breaks below first bottom (false breakdown)
  3. Price recovers above first bottom level
  4. Continue to neckline breakout

Entry: Two entry points:
  - Early: At false breakdown recovery (higher risk, higher reward)
  - Standard: At neckline breakout (confirmed)
Stop loss: Below false breakdown low
Target: Same as W bottom calculation
```

**Pattern 4: Bull Flag / Descending Pennant (下飘旗形)**
```
Detection:
  1. Strong upward wave (flag pole): swing low to swing high
  2. Pullback in descending channel (3+ bars, lower highs + lower lows)
  3. Pullback retraces 30-60% of flag pole (healthy)
  4. Breakout above flag's upper trendline
  5. Confirmation: breakout above the prior swing high

Target calculation:
  wave1_height = prior_high - prior_low
  pullback_low = lowest point of flag
  target = pullback_low + wave1_height

Entry: At breakout of flag upper trendline, or at breakout of prior high
Stop loss: Below flag lower trendline, or below pullback low
Volume: Volume should contract during flag, expand on breakout
```

**Pattern 5: Head & Shoulders Bottom (头肩底)**
```
Detection:
  1. Find 3 swing lows: left shoulder, head (lowest), right shoulder
  2. Head must be lower than both shoulders
  3. Shoulders approximately equal height (within 5%)
  4. Connect the two swing highs between shoulders = neckline
  5. Breakout above neckline = confirmed

Target calculation:
  distance = neckline - head_low
  target_wave1 = neckline + distance
  target_wave2 = target_wave1 + distance

Entry: At neckline breakout
Stop loss: Below right shoulder, or below neckline (5-7%)
Volume: 带量突破颈线 (breakout with volume)
Special: 破底翻 at the head increases confidence significantly
```

**Pattern 6: Symmetrical Triangle Bottom (收敛三角形底部)**
```
Detection:
  1. Converging trendlines: descending upper + ascending lower
  2. At least 4 touch points total (2 on each line)
  3. Breakout upward at 1/2 to 3/4 of triangle's horizontal length
  4. Breakout past 3/4 is less reliable (越靠尾端越容易失败)

Target calculation:
  triangle_side = highest_high - lowest_low (the left side of triangle)
  target = breakout_level + triangle_side

Entry: At upward breakout of upper trendline
Stop loss: Below lower trendline, or back below breakout level (5-7%)
Volume: Volume should decrease during triangle, spike on breakout
```

**Patterns 7-12: Bearish Mirrors**

Each bullish pattern has an exact bearish mirror:

| Bullish | Bearish Mirror | Key Difference |
|---------|---------------|----------------|
| W Bottom | **M Top (M头)** | Double top + neckline breakdown |
| False Breakdown | **False Breakout (假突破)** | Breaks above resistance then reverses below |
| False Breakdown W | **False Breakout H&S (假突破头肩顶)** | H&S where head is false breakout |
| Bull Flag | **Bear Flag (上飘旗形)** | Ascending channel pullback in downtrend |
| H&S Bottom | **H&S Top (头肩顶)** | Three peaks, head highest, neckline breakdown |
| Sym Triangle Bottom | **Sym Triangle Top (收敛三角形头部)** | Same triangle, breakdown instead of breakout |

The detection logic mirrors exactly — swap high/low, support/resistance, breakout/breakdown.

**Note on bearish patterns for stock screening:** Since the primary goal is finding stocks to BUY, bearish patterns serve as:
- **Avoidance signals** — don't buy a stock showing M top or H&S top
- **Exit signals** — if a held stock develops a bearish pattern, take profit
- The patterns are still detected and reported, but ranked separately

#### 3.4 Pattern Weight System (Configurable)

Different patterns have different historical reliability. The weight system is **configurable in settings.yaml** so you can tune it based on your own backtest results over time.

**Default pattern weights (from book's emphasis + empirical win rates):**

```python
# Higher weight = more reliable pattern, higher win rate historically
# These are configurable in config/settings.yaml
PATTERN_WEIGHTS: dict[str, float] = {
    # Tier 1: Highest reliability (蔡森 emphasizes these most in the book)
    "false_breakdown":          1.0,   # 破底翻 — book's favorite, "甩轿" shakes out weak hands
    "false_breakout":           1.0,   # 假突破 — "骗线", market makers' deception = strongest reversal
    "false_breakdown_w_bottom": 0.95,  # 破底翻+W底 — combination of two strong patterns

    # Tier 2: Very reliable classic patterns
    "w_bottom":                 0.85,  # W底 — classic double bottom, well-defined neckline
    "m_top":                    0.85,  # M头 — mirror of W bottom
    "hs_bottom":                0.80,  # 头肩底 — more complex but very reliable when clean
    "hs_top":                   0.80,  # 头肩顶 — mirror

    # Tier 3: Continuation patterns (good but need trend context)
    "bull_flag":                0.75,  # 下飘旗形 — strong in confirmed uptrends
    "bear_flag":                0.75,  # 上飘旗形 — mirror
    "false_breakout_hs":        0.75,  # 假突破+头肩顶 — complex, needs volume confirmation

    # Tier 4: Triangle patterns (direction uncertain until breakout)
    "sym_triangle_bottom":      0.65,  # 收敛三角形底部 — unreliable past 3/4 of triangle
    "sym_triangle_top":         0.65,  # 收敛三角形头部 — mirror
}
```

**Why 破底翻 and 假突破 are Tier 1:**
- The book explicitly says 破底翻 is "更进阶更安全的底部布局方式" (a more advanced, safer bottom entry method)
- 假突破 represents market maker deception (主力骗线) — when the "lie" is exposed, the reversal is violent and reliable
- Both patterns include a built-in stop loss level (the false low/high), making risk management clean
- The book shows these patterns working across decades of data (Microsoft 1993-2011, Apple 2007-2008)

**Why triangles are Tier 4:**
- 蔡森 explicitly warns: "收敛三角形需在1/2~3/4处跌破才算有效跌破，整理至越尾端的跌破，越容易产生失败"
- Direction is ambiguous until breakout occurs
- Higher false breakout rate compared to other patterns

#### 3.5 Pattern Confidence Scoring

Each detected pattern gets a confidence score 0.0-1.0, incorporating the pattern weight:

```python
def score_pattern(
    pattern: PatternSignal,
    vol_price: VolumePriceSignal,
    pattern_weights: dict[str, float]  # From config
) -> float:
    # Base score from pattern weight (0.0-1.0)
    weight = pattern_weights.get(pattern.pattern_type, 0.5)

    score = 0.0

    # Pattern weight contributes 35% of final score
    score += 0.35 * weight

    # Volume confirmation (蔡森's most important factor) — 25%
    if pattern.volume_confirmed:          # 带量突破
        score += 0.20
    if not vol_price.divergence:          # No 量价背离
        score += 0.05

    # Pattern clarity (how clean is the formation) — 15%
    # pattern_clarity_score is computed from measurable attributes:
    #   neckline_touch_count: 2 touches = 0.5, 3+ = 1.0
    #   symmetry: abs(left - right) / avg(left, right) → 0 is perfect, >0.2 is poor
    #   retracement_ratio: pullback within 30-60% of prior wave = 1.0, outside = 0.5
    #   weighted average of these three sub-scores
    pattern_clarity_score = compute_clarity(pattern)  # Returns 0.0-1.0
    score += 0.15 * pattern_clarity_score

    # Risk-reward — 15%
    if pattern.rr_ratio >= 3.0:
        score += 0.15
    elif pattern.rr_ratio >= 2.0:
        score += 0.10
    elif pattern.rr_ratio >= 1.5:
        score += 0.05

    # Status — 10%
    if pattern.status == "confirmed":
        score += 0.10
    elif pattern.status == "forming":
        score += 0.0  # Watch only

    # Pattern combination bonus (specific rules, max 0.1 total):
    combination_bonus = 0.0
    # 破底翻 + W底 structure detected together → +0.08
    # 假突破 at a pattern's peak (e.g., M top head is false breakout) → +0.08
    # Daily pattern confirmed AND weekly chart shows same direction → +0.05
    # Multiple bullish patterns on same stock (e.g., H&S bottom + bull flag) → +0.03
    combination_bonus = min(combination_bonus, 0.10)
    score += combination_bonus

    return min(score, 1.0)
```

#### 3.6 LLM Vision Validation Stage (AI Second Opinion)

After the rule-based detector finds pattern candidates, an LLM vision model validates them by "looking" at the rendered chart — the same way a human trader would.

**Architecture:**
```
Rule-based detector → candidates with rule_score >= 0.40
    ↓
Render candlestick chart as PNG (Plotly, already exists)
    ↓
Send chart image to Anthropic API (Claude Sonnet — fast + cheap + good vision)
    ↓
Claude rates pattern confidence 1-10
    ↓
Combined score = 0.6 * rule_score + 0.4 * llm_score
    ↓
Filter: combined >= threshold → include in report
```

**Implementation:**

```python
import anthropic
import base64
from pathlib import Path

client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from .env

def validate_pattern_with_llm(
    chart_png_path: Path,
    pattern_type: str,
    symbol: str,
    key_points: dict,  # e.g., {"neckline": 182.0, "bottom1": 170.0, "bottom2": 171.0}
) -> float:
    """Ask Claude to validate a detected pattern. Returns confidence 0.0-1.0."""
    image_data = base64.standard_b64encode(chart_png_path.read_bytes()).decode("utf-8")

    # Structured prompt with pattern context for better accuracy
    prompt = (
        f"This is a daily candlestick chart with volume for {symbol}.\n"
        f"My rule-based system detected a '{pattern_type}' pattern.\n"
        f"Key levels: {key_points}\n\n"
        f"Based on 蔡森's methodology (多空转折一手抓):\n"
        f"1. Is this a valid {pattern_type}? Look at the shape, symmetry, and neckline.\n"
        f"2. Does volume confirm? (带量突破 on breakout, 量价背离 at tops)\n"
        f"3. Rate confidence 1-10. Respond with ONLY the number."
    )

    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=50,
        messages=[{
            "role": "user",
            "content": [
                {"type": "image", "source": {
                    "type": "base64",
                    "media_type": "image/png",
                    "data": image_data,
                }},
                {"type": "text", "text": prompt},
            ],
        }],
    )
    score = int(response.content[0].text.strip()) / 10.0
    return score  # 0.0 - 1.0
```

#### 3.7 Daily QU Score (综合评分)

Each stock in QU100 gets a **daily composite score** combining all three layers:

```python
@dataclass
class DailyQUScore:
    symbol: str
    score_date: date

    # Component scores (each 0.0-1.0)
    money_flow_score: float     # Layer 1
    sector_score: float         # Layer 2
    pattern_score: float        # Layer 3 (best pattern's confidence)

    # Weighted composite
    composite_score: float      # Weighted sum below

    # Pattern details
    active_patterns: list[PatternSignal]
    pattern_weight_used: float  # The weight of the best pattern

    # Trend
    score_trend: str            # "improving", "stable", "declining" (vs yesterday)
    days_above_threshold: int   # Consecutive days composite > 0.55


# Composite formula (configurable weights):
LAYER_WEIGHTS = {
    "money_flow": 0.25,    # QU100 money flow signal
    "sector":     0.10,    # Sector trend bonus
    "pattern":    0.65,    # Technical pattern (dominant factor)
}

composite = (
    LAYER_WEIGHTS["money_flow"] * money_flow_score
    + LAYER_WEIGHTS["sector"] * sector_score
    + LAYER_WEIGHTS["pattern"] * pattern_score
)
```

**Score interpretation:**
| Composite Score | Rating | Action |
|----------------|--------|--------|
| 0.80 - 1.00 | S-tier | Strong buy — confirmed high-weight pattern + money flow + hot sector |
| 0.65 - 0.79 | A-tier | Buy — confirmed pattern, good money flow |
| 0.50 - 0.64 | B-tier | Watch — pattern forming or weak confirmation |
| 0.35 - 0.49 | C-tier | Hold off — insufficient signal |
| 0.00 - 0.34 | D-tier | Avoid — bearish signals or no pattern |

**Daily score persistence:**

Scores are stored daily in `stock_screen_results` so you can track score trends:
- A stock whose score jumps from C-tier to A-tier in one day (pattern just confirmed) = high priority alert
- A stock that's been B-tier for 5 days and ticking up = pattern forming, approaching confirmation
- A stock dropping from A to C = pattern failed, stop loss triggered

**Composite scoring** (uses same weights as Section 3.6 / settings.yaml — single source of truth):
```
composite = (0.25 * money_flow_score)    # from config: layer_weights.money_flow
          + (0.10 * sector_score)        # from config: layer_weights.sector
          + (0.65 * best_pattern_score)  # from config: layer_weights.pattern
```

Technical pattern confidence is weighted heaviest because that's the actual entry signal. Money flow is the filter that gets you to the right stocks. Sector trend is a bonus.

**Recommendation thresholds** (from config — single source of truth):
- `composite >= 0.80` → "strong_buy" (S-tier)
- `composite >= 0.65` → "buy" (A-tier)
- `composite >= 0.50` → "watch" (B-tier)
- `composite < 0.50` → "avoid" (C/D-tier)
